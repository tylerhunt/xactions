# Implementation Plan: Envelope Category Association

**Branch**: `005-envelope-categories` | **Date**: 2026-04-10 | **Spec**: [spec.md](spec.md)

## Summary

Add category association to envelopes: wire up the already-existing
`envelope_categories` join table to the create flow, and introduce an
edit flow triggered by a dropdown menu (ChevronDown trigger) next to
each envelope name in the Envelopes table.  The dropdown design is
taken directly from the Figma Make reference (`EnvelopeGrid.tsx`).

## Technical Context

**Language/Version**: Elixir 1.17+ / OTP 27  
**Primary Dependencies**: Phoenix 1.8.5, Phoenix LiveView 1.1.x, Tailwind CSS, `ecto_sqlite3`  
**Storage**: SQLite — `envelope_categories` join table already migrated  
**Testing**: ExUnit; integration tests hit the real SQLite DB  
**Target Platform**: Web (Phoenix LiveView, server-rendered)  
**Project Type**: Web application (single LiveView per page)  
**Performance Goals**: p95 < 200 ms for create/update envelope operations  
**Constraints**: Category–envelope assignment is exclusive: one category belongs to at most one envelope (enforced by unique constraint on `envelope_categories.category_id`)  
**Scale/Scope**: Single-user personal finance app; envelope count typically < 50
